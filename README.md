Энкрипто - симметричное шифрование

Encrypto - symmetric encryption

Шифруй сообщения и пароли — защити себя от взлома и дипфейков

# Длинное описание

Энкрипто — защити свои сообщения в один клик 🔐

# 🔐 Описание шифрования

Приложение обеспечивает безопасное шифрование текста с использованием современных криптографических стандартов. Оно сочетает простоту использования с надёжными алгоритмами защиты данных.


## 🛡️ Для пользователей

- 🔐 Сообщения шифруются с помощью **AES-256**
- 🔑 Секретный ключ **не хранится** — он создаётся из вашего пароля
- 🎲 Для каждого сообщения используется **уникальный случайный IV**
- 📦 Результат кодируется в **Base64** для удобного копирования и отправки

### Что это означает

- Прочитать сообщение может только тот, у кого есть ключ
- Одинаковые сообщения каждый раз шифруются по-разному
- Данные защищены от подмены и повреждения

---

## ⚙️ Технические детали

### Алгоритм

- **Алгоритм:** AES (Advanced Encryption Standard)
- **Режим:** GCM (Galois/Counter Mode)
- **Padding:** NoPadding
- **Длина ключа:** 256 бит
- **IV (nonce):** 12 байт (96 бит)
- **Тег аутентификации:** 128 бит

---

### 🔑 Генерация ключа

```text
password (UTF-8) → SHA-256 → 256-bit key
```

- Используется `SHA-256`
- Без соли (salt)
- Без итераций (не используется KDF)

---

### 🔐 Процесс шифрования

```text
Генерация IV (12 байт) → Инициализация AES/GCM → Шифрование → IV || ciphertext → Base64
```

**Формат результата:**

```text
Base64(IV[12 байт] + Ciphertext[n байт])
```

---

### 🔓 Процесс расшифровки

```text
Base64 → Извлечение IV → Извлечение ciphertext → Инициализация cipher → Расшифровка → plaintext
```

---

## 🔒 Безопасность

- ✔ Используется **AES-GCM (AEAD)**
- ✔ Обеспечивается конфиденциальность, целостность и аутентичность
- ✔ Случайный IV предотвращает одинаковый результат при одинаковых данных

---

## ⚠️ Ограничения

- ❌ Не используется salt при генерации ключа
- ❌ Нет усиления ключа (PBKDF2, Argon2)
- ❌ Безопасность зависит от сложности пароля

---

## 💡 Рекомендуемые улучшения

```text
password → PBKDF2 / Argon2 + salt + iterations → AES-256 key
```

---

## 📌 Итог

Текущая реализация использует надёжный алгоритм AES-GCM и подходит для безопасного обмена сообщениями. Для повышения уровня безопасности рекомендуется использовать KDF (например, PBKDF2 или Argon2).
